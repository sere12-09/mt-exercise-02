# MT Exercise 2: Pytorch RNN Language Models

This repo shows how to train neural language models using [Pytorch example code](https://github.com/pytorch/examples/tree/master/word_language_model). Thanks to Emma van den Bold, the original author of these scripts. 

# Requirements

- This only works on a Unix-like system, with bash.
- Python 3 must be installed on your system, i.e. the command `python3` must be available
- Make sure virtualenv is installed on your system. To install, e.g.

    `pip install virtualenv`

# Steps

Clone this repository in the desired place:

    git clone https://github.com/marcamsler1/mt-exercise-02
    cd mt-exercise-02

Create a new virtualenv that uses Python 3. Please make sure to run this command outside of any virtual Python environment:

    ./scripts/make_virtualenv.sh

**Important**: Then activate the env by executing the `source` command that is output by the shell script above.

Download and install required software:

    ./scripts/install_packages.sh

Download and preprocess data:

    ./scripts/download_data.sh

Train a model:

    ./scripts/train.sh

The training process can be interrupted at any time, and the best checkpoint will always be saved.

Generate (sample) some text from a trained model with:

    ./scripts/generate.sh



# MT Exercise 02 

In this MT exercise, I trained a recurrent neural network language model using the PyTorch example code provided in the repo: I analyzed how different dropout values influence the performance of the model.

I used the WikiText-2 dataset, which is already tokenized and split into training, validation and test sets.

The main change introduced in the code was the addition of a new argument `--log_file` in the `main.py` script. This allows the model to save training, validation and test perplexity values into a file during training: these log files are later used to create tables and plots for analysis, as described in the assignment.

To train a model, the following command was used:

python tools/pytorch-examples/word_language_model/main.py \
  --data tools/pytorch-examples/word_language_model/data/wikitext-2 \
  --dropout 0.2 \
  --emsize 200 \
  --nhid 200 \
  --epochs 5 \
  --log_file logs/dropout_0.2.tsv

I trained multiple models with different dropout values, including 0.0 (no dropout) and higher values up to 0.8: each model produces a separate log file stored in the `logs` directory.

Then, I created a `plot.py` script: this script reads all log files, extracts the perplexity values and generates line plots for both training and validation perplexity. In addition to that, it also prints the values in a format that can be used to construct tables in the PDF report, so that I would have it ready.


The plots can be generated by running:

python plot.py

This command creates two output files, `training_plot.png` and `validation_plot.png`, which can be seen in the final report.

Finally, we used the provided generation script to produce sample text from the trained models.


## Generated Text

To generate sample text from the trained model, the following command was used:

python tools/pytorch-examples/word_language_model/generate.py \
  --data tools/pytorch-examples/word_language_model/data/wikitext-2 \
  --checkpoint models/model.pt \
  --outf samples/sample.txt

This produces a file containing generated text based on the trained language model.



## Notes

The directories `logs/`, `models/` and `samples/` are not included in this repository, as they contain large or intermediate files generated during training and evaluation. The results derived from these files (tables and plots) are included in the final PDF report.
